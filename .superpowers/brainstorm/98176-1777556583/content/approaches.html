設計系統方案比較

方案 A

Token Only

只建立 CSS 變數（design token），不寫元件規則。頁面 import 後自行使用變數。

包含
CSS 變數 顏色 token 字型 token 間距 token 元件規則 ✗
最快完成（1-2天）
現有頁面改動最小
元件樣式仍然重複
新頁面還是要寫大量 CSS
工程量
⭐ 推薦 · 方案 B

Token + 核心元件

建立 token 系統，加上 6 個最常用元件。平衡速度和覆蓋率，立即可用於新頁面。

包含
CSS 變數 Nav Button Card Chip/Tag Form Input 展示頁
新頁面可快速組裝
token 改一次全部元件更新
展示頁讓維護更直觀
需要 3-4 天
工程量
方案 C

完整萃取

掃描全部 7 個頁面，把所有現有 pattern 都抽出來，同步重構現有頁面使用新系統。

包含
CSS 變數 全部元件 展示頁 重構現有頁面 Hero section Stats bar Blog card
最一致，技術債最少
現有頁面全部受益
工程量大，風險高
改動現有頁面可能引入 bug
工程量
